## Язык программирования

### Синтаксис (форма Бэкуса–Наура)

```
<program>       ::= <line>*
<line>          ::= <section_dir>
                  | <org_dir>
                  | <macro_def>
                  | <cond_block>
                  | <data_decl>
                  | <code_line>
                  | <label>
                  | <empty_line>

<section_dir>   ::= ".data" | ".text"
<org_dir>       ::= ".org" <number>
<macro_def>     ::= "%define" <name> <number>
<cond_block>    ::= "%ifdef" <name> <line>* ["%else" <line>*] "%endif"

<data_decl>     ::= [<label>] <data_dir>
<data_dir>      ::= ".num" <imm_val>
                  | ".pstr" '"' <chars> '"'

<code_line>     ::= [<label>] <instr>
<label>         ::= <name> ":"
<instr>         ::= <opcode> [<imm_val>]
<imm_val>       ::= <number> | <ident>      ; ident resolves to label address or macro value
<number>        ::= ["-"] (decimal | "0x" hex | "0o" octal | "0b" binary)
```

Комментарий: всё от `;` до конца строки.

### Семантика

**Стратегия вычислений** - строго последовательная (императивная). Каждая инструкция выполняется полностью до начала следующей. Порядок выполнения определяется счётчиком команд `PC`; переходы (`JMP`, `JZ`, `JNZ`, `JO`, `CALL`) изменяют `PC`.

**Области видимости** - глобальные. Метки (`label:`) видны во всём файле после первого прохода транслятора. Макросы (`%define`) также глобальны и разворачиваются до трансляции в машинный код.

**Типизация** - отсутствует. Все значения - 32-битные знаковые целые числа. Символы хранятся как `ord(c)`.

**Литералы:**
- Числа: десятичные, шестнадцатеричные (`0x…`), восьмеричные (`0o…`), двоичные (`0b…`). Литерал помещается непосредственно в операнд инструкции (24-битный знаковый диапазон: −8 388 608 … +8 388 607). Числа вне этого диапазона или используемые многократно должны размещаться в секции `.data` директивой `.num`.
- Строки: хранятся только в секции `.data` как Pascal-строка - первое машинное слово содержит длину, следующие слова - символы по одному (UTF-8 код символа, одно слово на символ).

**Константы** (`%define NAME VALUE`): разворачиваются транслятором на этапе препроцессинга. Памяти не занимают.

**Переменные**: объявляются директивой `.num` в секции `.data`. Каждая занимает одно 32-битное машинное слово.

**Процедуры**: вызываются инструкцией `CALL addr`. Адрес возврата сохраняется в Return Stack. Завершаются инструкцией `RET`.

**Прерывания**: вектор прерывания задан по адресу `0x000`. При прерывании аппаратно сохраняется `PC` в Return Stack и сбрасывается флаг `EI`. Завершается инструкцией `IRET` (восстанавливает `PC` и `EI`).

**Пример программы:**
```asm
%define OUTPUT_ADDR_SYMB 2046

.data
message: .pstr "Hello!"
len: .num 0
ptr: .num 0

.text
_start:
    push_m message
    pop_m len
    push message
    push 1
    add
    pop_m ptr
loop:
    push_m len
    jz end
    push OUTPUT_ADDR_SYMB
    push_m ptr
    push_ind
    pop_ind
    push_m ptr
    push 1
    add
    pop_m ptr
    push_m len
    push 1
    sub
    pop_m len
    jmp loop
end:
    halt
```

## Организация памяти

### Модель памяти

Архитектура фон Неймана: единое адресное пространство для инструкций и данных.

```
     Адрес       Описание
  +-----------+----------------------------------+
  | 0x000     | JMP <isr> (опционально)          | <- вектор прерывания
  | ...       |                                  |
  | M         | начало данных (.data)            | <- переменные, строки, числа
  | ...       | данные                           |
  | N (_start)| начало кода (.text)              | <- точка входа
  | ...       | инструкции                       |
  | 2045      | MMIO: INPUT_ADDR                 | <- read
  | 2046      | MMIO: OUTPUT_ADDR_SYMB           | <- write (ASCII-вывод)
  | 2047      | MMIO: OUTPUT_ADDR_DEС            | <- write (вывод числа)
  +-----------+----------------------------------+
```

**Размер:** 2048 машинных слов по 32 бита каждое (2045 из них доступно под код и данные).

**Есть 2 стека:**
- `Data Stack` - стек данных (глубина до 256). Все вычисления производятся только через него.
- `Return Stack` - стек возвратов (глубина до 256); хранит адреса возврата для `CALL`/`RET`/`IRET`. Программист не управляет им напрямую.

**Виды адресации:**
| Вид | Пример | Описание |
|---|---|---|
| Непосредственная | `PUSH 42` | Число в операнде инструкции (24 бита, знаковое) |
| Прямая | `PUSH_M addr` | Адрес в операнде, читается `MEM[addr]` |
| Косвенная | `PUSH_IND` | Адрес на вершине стека, читается `MEM[T]` |

**Отображение программы и данных на память:**
- Метки-константы (`%define`): только подстановка при трансляции, в памяти не сохраняются.
- `.num N`: одно слово в памяти с начальным значением N.
- `.pstr "str"` -> `len(str)+1` слов: сначала длина, потом символы по одному слову.
- Инструкции: одно 32-битное слово (8 бит опкод + 24 бита операнд).
- Адрес прерывания 0x000 (задан жестко): туда должен быть помещён `JMP` на обработчик.

## Система команд

### Особенности процессора

- **Архитектура:** стековая (`stack`). Нет регистров общего назначения - все вычисления ведутся через `Data Stack`. Return Stack аппаратно обслуживает `CALL`/`RET`/`IRET`.
- **Ввод-вывод:** memory-mapped IO (адреса ячеек 2045–2047). Доступ инструкциями `PUSH_M` / `PUSH_IND` / `POP_M` / `POP_IND`.
- **Прерывания:** флаг `IRQ` вычисляется каждый такт как `len(input_buffer) != 0`. При `IRQ & EI`: `RS.push(PC); PC <- 0x000; EI <- 0`. `IRET` восстанавливает PC и устанавливает `EI <- 1`.
- **Прерывание во время прерывания** невозможно (EI = 0). Если символ поступил пока EI = 0, он накапливается в `input_buffer`. Новая обработка прерывания запускается перед выполнением первой инструкции после IRET`.
- **Флаги:**
  - `carry` (C) - перенос/заём при беззнаковой арифметике:
    - ADD: устанавливается, если результат не вмещается в 32 бита 
    - SUB: устанавливается, если уменьшаемое меньше вычитаемого (беззнаково)
    - ADC, SBC: обновляются по тем же правилам, дополнительно учитывая старый carry во входных данных 
  - overflow (V) - знаковое переполнение:
    - ADD, ADC: устанавливается, если оба операнда одного знака, а результат другого 
    - SUB, SBC: устанавливается, если операнды разных знаков, а результат не того знака, что уменьшаемое 
    - DIV: устанавливается при делении INT_MIN на -1
  
  Инструкции ADD, SUB, ADC и SBC перезаписывают оба флага. Остальные инструкции флаги не трогают.
### Кодирование инструкций

Каждая инструкция - одно 32-битное машинное слово:

```
 31      24 23                   0
+----------+----------------------+
|  opcode  |       operand        |
| (8 бит)  |  (24 бита, знаковое) |
+----------+----------------------+
```

Операнд знаково-расширяется до 32 бит при декодировании.

### Набор инструкций

Указаны такты на полный цикл: 1 (fetch) + такты execute.
У`PUSH_IND`/`POP_IND` execute происходит за 2 такта из-за дополнительного обращения к памяти данных по косвенному адресу.

| Мнемоника | Опкод | Операнд | Действие                                        | Тактов |
|---|---|---|-------------------------------------------------|---|
| `PUSH`     | 0x01 | `imm`  | `DS.push(imm)`                                  | 2 |
| `PUSH_M`   | 0x02 | `addr` | `DS.push(MEM[addr])`                            | 2 |
| `PUSH_IND` | 0x03 | -      | `DS.push(MEM[DS.pop()])`                        | 3 |
| `POP`      | 0x04 | -      | `DS.pop()`                                      | 2 |
| `POP_M`    | 0x05 | `addr` | `MEM[addr] = DS.pop()`                          | 2 |
| `POP_IND`  | 0x06 | -      | `val=DS.pop(); addr = DS.pop(); MEM[addr] = val` | 3 |
| `DUP`      | 0x07 | -      | `DS.push(DS[-1])`                               | 2 |
| `ADD`      | 0x09 | -      | `DS.push(DS.pop() + DS.pop())` ; C, V           | 2 |
| `SUB`      | 0x0A | -      | `DS.push(DS.pop() − DS.pop())` ; C, V           | 2 |
| `MUL`      | 0x0B | -      | `DS.push((DS.pop() × DS.pop()) & 0xFFFFFFFF)`   | 2 |
| `MULH`     | 0x0C | -      | `DS.push((DS.pop() × DS.pop()) >> 32)`          | 2 |
| `ADC`      | 0x0D | -      | `DS.push(DS.pop() + DS.pop() + C)` ; C, V       | 2 |
| `SBC`      | 0x0E | -      | `DS.push(DS.pop() − DS.pop() − C)` ; C, V       | 2 |
| `DIV`      | 0x0F | -      | `DS.push(DS.pop() ÷ DS.pop())` (целочисленное)  | 2 |
| `MOD`      | 0x10 | -      | `DS.push(DS.pop() % DS.pop())`                  | 2 |
| `CMP`      | 0x11 | -      | `DS.push(DS.pop() == DS.pop() ? 1 : 0)`         | 2 |
| `GT`       | 0x12 | -      | `DS.push(DS.pop() > DS.pop() ? 1 : 0)`          | 2 |
| `LT`       | 0x13 | -      | `DS.push(DS.pop() < DS.pop() ? 1 : 0)`          | 2 |
| `JMP`      | 0x14 | `addr` | `PC = addr`                                     | 2 |
| `JZ`       | 0x15 | `addr` | `if DS.pop() == 0: PC = addr`                   | 2 |
| `JNZ`      | 0x16 | `addr` | `if DS.pop() != 0: PC = addr`                   | 2 |
| `JO`       | 0x17 | `addr` | `if OV: PC = addr`                              | 2 |
| `CALL`     | 0x18 | `addr` | `RS.push(PC); PC = addr`                        | 2 |
| `RET`      | 0x19 | -      | `PC = RS.pop()`                                 | 2 |
| `IRET`     | 0x1A | -      | `PC = RS.pop(); EI = 1`                         | 2 |
| `HALT`     | 0x1B | -      | остановка                                       | 2 |

## Транслятор

### Интерфейс командной строки

```
python3 src/translator.py <исходный.asm> <выходной.bin>
```

Создаёт два файла:
- `<выходной.bin>` - бинарный файл. Первые 4 байта - начальный адрес (`_start`), далее 2048 × 4 байта памяти.
- `<выходной_dump.log>` - текстовый дамп в формате `<addr> - <HEXCODE> - <mnemonic>`.

Пример:
```
$ python3 src/translator.py examples/hello.asm binAndLog/hello.bin
```

Пример дампа:
```
START: 0001

0000 - 14000002 - JMP 2
0001 - 14000001 - JMP 1
0002 - 020007FD - PUSH_M 2045
0003 - 07000000 - DUP
0004 - 01000000 - PUSH 0
0005 - 11000000 - CMP
0006 - 15000008 - JZ 8
0007 - 1B000000 - HALT
0008 - 050007FE - POP_M 2046
0009 - 1A000000 - IRET
0010-2047 - 00000000 - 0
```

### Этапы трансляции

```
Исходный код (.asm)
      |
      V
Первый проход: препроцессинг + разметка
Подстановка %define, условная компиляция (%ifdef / %else / %endif),
разбор .data / .text / .org, назначение адресов меткам,
запись данных (.num, .pstr) в массив памяти,
сбор списка инструкций с их адресами (опкод + операнд)
      |
      V
Второй проход: кодирование
Подстановка меток и макросов в операндах,
кодирование каждой инструкции в 32-битное слово,
запись в массив памяти
      |
      V
Запись .bin и _dump.log
```

**Ограничения транслятора:**
- Нет арифметики в макросах (только замена на число).
- Нет вложенности `%ifdef`.
- Операнды: только метки, числа или имена макросов (нет сложных выражений).
- Нет проверки типов или диапазонов операндов.

## Модель процессора

### DataPath

![datapath.svg](scheme/datapath.svg)

**Сигналы (управляются ControlUnit):**

| Сигнал      | Описание                                                                               |
|-------------|----------------------------------------------------------------------------------------|
| `ds_push`   | Пушит выход MUX в Data Stack                                                           |
| `ds_pop`    | Делает поп T из Data Stack                                                             |
| `addr_sel`  | Селектор для выбора значения, которое пойдет в AR                                      |
| `latch_ar`  | Защелкивает Address Register из MUX                                                    |
| `dr_sel`    | Селектор для выбора значения, которое пойдет в DR                                      |
| `latch_dr`  | Защелкивает Data Register                                                              |
| `alu_op`    | Код операции для ALU                                                                   |
| `stack_sel` | Селектор для выбора значения, которое пойдет в T (Data Stack)                          |
| `read`      | Инициирует чтение: выбранное устройство (память или интерфейс) выдает данные на шину   |
| `write`     | Инициирует запись: выбранное устройство (память или интерфейс) принимает данные с шины |

### ControlUnit

![controlunit.svg](scheme/controlunit.svg)


**Сигналы ControlUnit:**

| Сигнал | Описание                                            |
|---|-----------------------------------------------------|
| `pc_sel` | Селектор для выбора значения, которое пойдет в PC   |
| `latch_pc` | Защелкивает PC                                      |
| `rs_push` | Пушит PC в Return Stack                             |
| `rs_pop` | Делает поп из Return Stack                          |
| `latch_ir` | Защелкивает IR                                      |
| `inc` | Увеличивает Step Counter на 1                       |
| `reset` | Обнуляет Step Counter                               |

**Такт выборки (общий для всех инструкций):**
`addr_sel = PC`, `latch_ar` -> AR <- PC; `read`, `latch_ir` -> IR <- MEM[AR]; PC <- PC + 1

**Пример PUSH_M addr (1 такт исполнения):**
- `addr_sel = imm`
- `latch_ar`, AR <- operand
- `read`
- `latch_dr`, DR <- MEM[AR]
- `stack_sel = DR`
- `ds_push`

**Пример POP_IND (2 такта исполнения):**
1. Такт 1
- `dr_sel = T`
- `latch_dr` -> DR <- T
2. Такт 2
- `ds_pop`, T = addr
- `addr_sel = T`
- `latch_ar` -> AR <- T
- `write` -> MEM[AR] <- DR


### Особенности реализации симулятора

- Точность - такт. Большинство инструкций: **2 такта**. `PUSH_IND`, `POP_IND`: **3 такта**.
- Два аппаратных стека: `data_stack` и `return_stack`. Переполнение -> исключение -> останов.
- MMIO: тип доступа определяется адресом. `memory_read(2045)` -> pop из `input_buffer`. `memory_write(2046, v)` -> ASCII в `output_buffer`. `memory_write(2047, v)` -> decimal в `output_buffer`.
- IRQ: `self.irq = bool(self.dp.input_buffer)`. Это гарантирует, что все символы, пришедшие пока прерывания были запрещены, будут обработаны.

## Тестирование

### Инструментальная цепочка

```
<name>.asm
       |  python3 src/translator.py <name.asm> <output.bin>
       V
<name>.bin + <name>_dump.log
       |  python3 src/machine.py <binary.bin> [trap_schedule.txt]
       V
stdout: журнал тактов + итоговая строка "Output: …"
```

**Пример:**
```
$ python3 src/translator.py examples/hello.asm tmp/hello.bin

$ python3 src/machine.py /tmp/hello.bin
 Tick: 0000 | pc: 0010 | DS: [] | EI: 1 | carry: False | Exec: PUSH_M 0
 Tick: 0001 | pc: 0011 | DS: [13] | EI: 1 | carry: False | Exec: POP_M 14
 ...
Output: 10 + 20 + 30 + 40 + 50 = 150
Overall quantity of ticks: 248
Instructions executed: 217
```

### Golden-тесты

Тесты расположены в `tests`. Запуск:

```
pytest                           # обычный запуск
pytest tests/ --update-golden    # перегенерировать эталоны
```

| Тест          | Алгоритм                                                                    |
|---------------|-----------------------------------------------------------------------------|
| `hello`       | Напечатать hello world                                                      |
| `cat`         | Печатать данные, поданные через ввод                                        |
| `hello_user`  | Запросить у пользователя его имя, считать его, вывести на экран приветствие |
| `sort`        | Пузырьковая сортировка массива                                              |
| `double_math` | 64-битная арифметика                                                        |
| `prob2`       | Euler #6: разность квадрата суммы и суммы квадратов 1..100                  |
| `array_sum`   | Сумма элементов массива                                                     |

**Структура golden-файлов:**

```
tests/golden/<name>.yml
    in_source - код алгоритма
    output - вывод без журнала
    machine_code - машинный код и данные
    out_log - лог
    in_stdin (опционально) - входные данные
```
